## 本書について

**GitHub Organizational Operations & Principles Guide** は、  
GitHub の導入から運用、進化までをカバーする「原則（Bible）」と「運用指針（Framework）」をまとめたハンドブックです。  

公式ドキュメントを補完しつつ、現場で直面しやすい課題や「迷子になりがちなポイント」に焦点を当てます。  

---

## 本書が扱うもの

- **原則（Principles / Bible）**  
  ガバナンス、安全性、スピード、アジリティ、持続可能性 ― なぜこれらが重要か。

- **運用（Framework / Operations）**  
  Org/Team 設計、ワークフロー設計、Copilot 段階導入などの具体的アプローチ。

- **Do & Don’t**  
  実務で得られた「やるべきこと」「やってはいけないこと」の併記。

---

## スコープ

1. **導入（Adoption）**  
   GitHub や Copilot を心理的安全性を確保しつつ、段階的に導入する。

2. **運用（Operations）**  
   日々の開発を支えるガバナンス・コンプライアンス・セキュリティ。

3. **進化（Evolution）**  
   小規模チームから大規模 Enterprise への拡張、新機能の取り込みと改善サイクル。
